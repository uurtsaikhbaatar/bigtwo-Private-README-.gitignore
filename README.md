# Дай Ди (Big Two)

Найзуудтайгаа онлайнаар тоглох хятад покерийн апп. Нэг нь өрөө үүсгээд 6 оронтой
кодоо илгээхэд бусад нь нэгдэнэ. iOS болон Android дээр ажиллана.

- **App** — Expo (React Native), TypeScript. iOS, Android, вэб гурвуулан дээр ижил код.
- **Server** — Node.js + WebSocket (`ws`). Тоглоомын бүх дүрэм сервер дээр шалгагдана.
  Мөн вэб хувилбарыг өөрөө дамжуулдаг тул нэг л хаяг байрлуулахад хангалттай.
- **Хуваалцсан код** — `app/src/shared/` дотор. Апп болон сервер яг ижил логикийг
  ашигладаг тул дүрэм хоёр тал дээр зөрөх боломжгүй.

App Store-д тавихгүй, Mac ч шаардлагагүй — найзууд **вэб хаягаар** ороод
Safari-гийн "Home Screen-д нэмэх"-ийг дарахад жинхэнэ апп шиг ажиллана.

## Бүтэц

```
bigtwo/
├── app/                  Expo апп
│   └── src/
│       ├── shared/       ★ тоглоомын логик (сервер ч эндээс импортолдог)
│       │   ├── cards.ts      хөзрийн дүрслэл, тараалт
│       │   ├── combos.ts     хослол таних ба харьцуулах
│       │   ├── game.ts       төлөвийн машин, дүрмийн шалгалт, оноо
│       │   └── protocol.ts   клиент↔сервер мессеж, харагдац
│       ├── components/   PlayingCard, Button
│       ├── screens/      Home, Lobby, Table
│       ├── net.ts        WebSocket клиент (авто-reconnect)
│       └── storage.ts    нэр / сервер / суудал хадгалах
│   └── public/           PWA manifest, дүрс, index.html загвар
└── server/               WebSocket сервер, өрөөний менежмент, статик файл
```

## Найзуудтайгаа тоглох (үндсэн зам)

```bash
npm run setup      # хамаарлуудыг суулгах (эхний удаад)
npm run build:web  # вэб хувилбарыг бүтээх → app/dist
npm run server     # сервер асаах (0.0.0.0:8787)
```

Дараа нь найзууд браузераасаа `http://<таны-IP>:8787` хаягийг нээнэ
(IP-гээ `ipconfig`-оос харна). Энэ нь **ижил Wi-Fi** дээр ажиллана.

iPhone дээр Safari → Хуваалцах товч → **"Home Screen-д нэмэх"**. Дараа нь хаяг
харагдахгүй, бүтэн дэлгэцээр, жинхэнэ апп шиг нээгдэнэ.

## Найзуудаа хэрхэн урих вэ

Өрөө үүсгэхэд лоббид **"Найзууддаа илгээх"** товч гарна. Тэр нь өрөөний кодыг
агуулсан бүтэн линк үүсгэдэг:

```
https://таны-хаяг/?code=3W5LT9
```

Найз тэр линк дээр дарахад:

| Тохиолдол | Юу болох вэ |
|---|---|
| Анх удаа | "Танийг 3W5LT9 өрөөнд урьжээ" гэж гарч, код нь бөглөгдсөн байна — зөвхөн нэрээ бичнэ |
| Өмнө тоглож байсан | Нэрийг нь санаж байгаа тул **шууд өрөөнд орно**, товч дарахгүй |
| Өрөө нь дууссан | Нүүр хуудсанд тайлбартай үлдэж, шинэ код оруулах боломжтой |

Өрөөнд орсны дараа хаягнаас `?code=` хэсэг автоматаар арилдаг тул хуудсыг дахин
ачаалахад хуучирсан кодоор оролдохгүй.

## Интернэтэд байршуулах

Найзууд өөр газраас тоглох бол серверийг үүлэн дээр гаргана. `Dockerfile` нь вэб
хувилбарыг бүтээж, сервертэй нэг image болгодог тул нэг л команд хангалттай.

### Render — үнэгүй (санал болгож буй)

`render.yaml` бэлэн байгаа тул CLI хэрэггүй — бүгд вэбээр хийгдэнэ.

```bash
# 1. Кодоо GitHub дээр гаргах (bigtwo хавтас нь repo-гийн үндэс байх нь тохиромжтой)
cd bigtwo
git init && git add . && git commit -m "Дай Ди"
gh repo create bigtwo --private --source=. --push
```

2. [dashboard.render.com](https://dashboard.render.com) → **New → Blueprint**
3. GitHub repo-гоо сонгоно. Render `render.yaml`-ыг уншиж тохиргоог өөрөө хийнэ.
4. **Apply** дарна. Эхний build ~5 минут (вэб хувилбарыг бүтээдэг тул).

Дуусахад `https://bigtwo-xxxx.onrender.com` хаяг гарна — тэрийгээ найзууддаа
илгээнэ. GitHub-д push хийх бүрд автоматаар шинэчлэгдэнэ.

**Үнэгүй багцын нөхцөл:**

| Зүйл | Утга |
|---|---|
| Унтрах | 15 минут **огт хөдөлгөөнгүй** байвал |
| Сэрэх | ~50 секунд (эхний хүн хүлээнэ) |
| Тоглож байхад | WebSocket мессеж нь "хөдөлгөөн"-д тооцогддог тул **унтрахгүй** |
| Хязгаар | Сард 750 цаг — нэг үйлчилгээнд хангалттай |

Өөрөөр хэлбэл: эхний хүн орохдоо ~50 секунд хүлээж магадгүй, гэхдээ тоглоом
эхэлсэн хойно тасрахгүй. Хэн ч тоглоогүй үед унтардаг тул идэвхгүй өрөө алга
болно — тоглогчид холбогдсон байхад асуудалгүй.

### Fly.io — унтардаггүй, гэхдээ төлбөртэй

`fly.toml` мөн бэлэн (Токиогийн бүс, Монголд хамгийн ойр). Хүлээлтгүй байлгах
бол энэ нь дээр — жижиг машин сард ойролцоогоор $2–3.

```bash
iwr https://fly.io/install.ps1 -useb | iex   # CLI суулгах (PowerShell)
fly auth signup
cd bigtwo
fly launch --copy-config --no-deploy         # аппын нэрээ сонгоно
fly deploy
```

Railway ч ижил `Dockerfile`-ыг таньдаг ($5 туршилтын кредит, дараа нь төлбөртэй).

> Өрөөнүүд зөвхөн санах ойд хадгалагддаг тул сервер дахин ачаалахад идэвхгүй
> өрөө алга болно. Тоглогчид холбогдсон байх хугацаанд асуудалгүй.

## Хөгжүүлэлт

```bash
npm run server    # 1-р цонх: сервер
npm run app       # 2-р цонх: Expo (утсан дээр Expo Go-гоор QR уншина)
```

Expo Go ашиглах үед апп серверийн хаягийг Expo-гийн хаягаас автоматаар таана.
Өөр хаяг руу заах бол: Нүүр хуудас → **Сервер тохиргоо** → `ws://…:8787`

> Нэг компьютер дээр хэд хэдэн тоглогчоор турших бол вэб хувилбарыг өөр өөр табд
> нээнэ — суудал таб тус бүрд тусдаа хадгалагддаг.

## Дүрэм (энэ хувилбарт хэрэгжсэнээр)

### Хөзөр тавих

- Зэрэглэл: `3 < 4 < … < K < A < 2`. Баг: `♦ < ♣ < ♥ < ♠`.
- Хослолууд: ганц, хос, гурвал, 5 хөзөр.
  5 хөзрийн эрэмбэ: шулуун < өнгө < дүүрэн байшин < дөрвөл < шулуун өнгө.
- Ширээн дээрхтэй **ижил тооны** хөзрөөр л дарна.
- Бүгд пас хийвэл сүүлд тавьсан хүн шинэ эргэлт эхлүүлнэ.
- Шулуун нь `3…K-A-2` дарааллаар явна. Мөн **`A-2-3-4-5` (wheel)** хүчинтэй бөгөөд
  покерын нэгэн адил **хамгийн сул шулуунд** тооцогдоно. `2-3-4-5-6` нь шулуун биш.

### Хэн эхлэх вэ

| Нөхцөл | Эхлэгч |
|---|---|
| **≤4 тоглогч**, эхний дугуй | **3♦**-тэй тоглогч |
| ≤4 тоглогч, дараагийн дугуйнууд | Өмнөх дугуйн **хожигч** |
| **5+ тоглогч**, дугуй бүрд | **3♦**-тэй тоглогч (хожигч өнждөг тул) |
| Хасалтаар 5→4 болсон дугуй | **3♦**-тэй тоглогч — **сүүлийн удаа** |
| Түүнээс хойш | Хожигч |

3♦ нь зөвхөн **хэн эхлэхийг** заана — эхлэгч дуртай хослолоо тавьж болно,
3♦-ээ заавал оруулах шаардлагагүй.

3♦ хэн нэгэнд тарааагдаагүй бол (2–3 тоглогчтой үед 52 хөзөр бүгд
тарааагддаггүй) хамгийн бага хөзөртэй тоглогч эхэлнэ.

### Оноо ба хожил

Дугуй бүрийн эцэст үлдсэн хөзрөөр оноо **нэмэгдэнэ** — оноо хэзээ ч буурахгүй.

| Үлдсэн хөзөр | Үржүүлэгч | Жишээ |
|---|---|---|
| 1–9 | ×1 | 5 хөзөр → +5 |
| 10–12 | ×2 | 12 хөзөр → +24 |
| 13 (нэг ч хөзөр гаргаагүй) | ×3 | 13 хөзөр → +39 |

Тоглоом эхлэхэд **босго оноог** (30 эсвэл 40) сонгоно. Босгод хүрсэн тоглогч
**хасагдаж**, бусад нь үргэлжлүүлнэ. Хамгийн сүүлд үлдсэн тоглогч хожино.

### 🐉 Луу

Тараалтад хэн нэгэнд **13 зэрэглэл бүрээс нэг хөзөр** (баг хамаарахгүй) буувал тэр
тоглогч оноо ямар ч байсан **тоглолтыг шууд хожино**, бусад бүгд хожигдоно.

### 5–8 тоглогч

Хөзөр 52 ширхэг тул нэг дугуйд зөвхөн **4 тоглогч** сууна. Илүү хүн байвал:

1. **Эхний дугуй** — бүгд нэг хөзөр сугалж, хамгийн бага 4 нь тоглоно.
2. **Дараагийн дугуйд** — өмнөх дугуйн 1-р (болон 2-р) байр эзэлсэн нь өнжиж,
   өнжсөн хүмүүсээс хамгийн бага хөзөр сугалсан нь ордог.
3. **5 тоглогчтой** үед өнжигч ганц тул дугуй бүрд 1 хүн солигдоно;
   6 ба түүнээс дээш үед 2 хүн солигдоно.

Өнжиж буй тоглогч **ширээн дээрх хөзрийг харна**, гэхдээ хэний ч гарыг харахгүй
бөгөөд оноо нь нэмэгдэхгүй.
